# Challenge Guide: GitHub CoPilot Java App Modernization

### Estimated Duration: 120 Minutes 

## Introduction

The legacy **Asset Manager** application is a Java-based web application that currently relies on **local PostgreSQL**, **AWS S3 for storage**, and **RabbitMQ for messaging**. The application is not cloud-native, uses older Java and Spring frameworks, and is not containerized for modern deployment scenarios.

In this challenge, you will **assess, modernize, migrate, containerize, and deploy** the application using the **GitHub Copilot App Modernization extension**. You will begin by running and analyzing the application, upgrading the Java runtime and frameworks, and then migrating the database, storage, and messaging services to **Azure native services**. Next, you will enable application health monitoring using **Spring Boot Actuator**, containerize the application using **Docker**, and finally deploy the modernized solution to **Azure Kubernetes Service (AKS)**.

This challenge provides hands-on experience with **AI-assisted modernization**, **cloud migration**, **containerization**, and **Azure deployment automation**.

* **GitHub Copilot App Modernization:** An AI-powered extension that analyzes, modernizes, migrates, and deploys applications using predefined and custom automation tasks.
* **Azure Database for PostgreSQL Flexible Server:** A fully managed PostgreSQL service for secure, scalable relational data.
* **Azure Blob Storage:** Microsoft’s object storage solution for unstructured data such as images and files.
* **Azure Service Bus:** A fully managed enterprise messaging service for reliable communication between application components.
* **Spring Boot Actuator:** A monitoring tool that provides health, metrics, and readiness endpoints for Spring Boot applications.
* **Docker:** A container platform used to package the application and its dependencies into portable containers.
* **Azure Kubernetes Service (AKS):** A managed Kubernetes service for deploying and managing containerized applications at scale.


## Challenge Objectives

### 1. Assess the Java Application

- Launch Docker Desktop and ensure the Docker engine is running.
- Navigate to the **asset-manager** project folder and start the application using the provided startup script.
- Access the application UI, RabbitMQ management portal, and PostgreSQL services locally.
- Open the project in **Visual Studio Code**.
- Use the **GitHub Copilot App Modernization** extension to run the **application assessment**.
- Review the generated **Assessment Report** and analyze issues, framework versions, and migration blockers.

### 2. Upgrade Java Runtime and Frameworks

- From the assessment report, run the **Java Version Upgrade** predefined task.
- Allow the Copilot agent to:

  - Upgrade the JDK version
  - Update Spring and Spring Boot frameworks
- Review and apply the generated changes.

### 3. Migrate to Azure Database for PostgreSQL

- Run the predefined task to **migrate from local PostgreSQL to Azure Database for PostgreSQL Flexible Server**.
- Review the autogenerated **migration plan**.
- Allow Copilot to:

  - Update application configuration
  - Provision Azure PostgreSQL resources
  - Apply required code changes
- Validate successful migration.

### 4. Migrate to Azure Blob Storage

- Run the predefined task to **replace AWS S3 with Azure Blob Storage**.
- Allow Copilot to:

  - Update storage configurations
  - Provision Blob Storage in Azure
  - Modify application code as required
- Validate storage migration.

### 5. Migrate to Azure Service Bus

- Run the predefined task to **migrate from RabbitMQ to Azure Service Bus**.
- Allow Copilot to:

  - Refactor messaging components
  - Update configurations and dependencies
  - Provision Azure Service Bus resources
- Validate messaging migration.


### 6. Expose Health Endpoints Using a Custom Task

- Create a **custom Copilot task** to enable **Spring Boot Actuator health endpoints**.
- Add the official Spring Boot Actuator documentation `https://docs.spring.io/spring-boot/reference/actuator/endpoints.html` as a reference.
- Run the custom task and apply the generated changes.
- Verify that health endpoints are successfully enabled.

### 7. Containerize the Application

- Run the predefined **Containerize Application** task.
- Allow Copilot to:

  - Generate Dockerfiles
  - Build container images
  - Fix build errors
- Verify successful container image creation.

### 8. Deploy the Application to Azure

- Run the predefined deployment task:
  **Provision Infrastructure and Deploy to Azure**
- Modify the deployment target to **Azure Kubernetes Service (AKS)**.
- Allow Copilot to:

  - Generate an Azure deployment plan
  - Provision required Azure resources
  - Execute deployment scripts
- Validate:

  - Pods are running in AKS
  - Application is accessible via the external IP
  - All deployed resources are visible in the Azure Portal


##  Success Criteria

- You successfully ran the Java application locally.
- You generated and reviewed the modernization **assessment report**.
- You upgraded the Java runtime and Spring frameworks.
- You migrated:

  - Database to **Azure Database for PostgreSQL Flexible Server**
  - Storage to **Azure Blob Storage**
  - Messaging to **Azure Service Bus**
- You enabled **Spring Boot Actuator health endpoints**.
- You containerized the application using **Docker**.
- You deployed the application successfully to **Azure Kubernetes Service (AKS)**.
- You validated all Azure resources in the **Azure Portal**.
